## Reporting Bugs

<Steps>
  <Step title="Search Existing Issues">
    Before creating a new issue, search [existing issues](https://github.com/Seyamalam/bun-scikit/issues) to see if your bug has already been reported.
  </Step>
  
  <Step title="Use the Bug Report Template">
    Use the bug report issue template to ensure you provide all necessary information.
  </Step>
  
  <Step title="Include Required Information">
    Make sure to include:
    <Accordion title="Required Bug Report Information" icon="list-check">
      - **Minimal reproduction code**: A complete, minimal example that demonstrates the issue
      - **Input data shape/sample**: Description or sample of the data being used
      - **Bun version**: Output of `bun --version`
      - **bun-scikit version**: Version number from `package.json` or `bun pm ls`
      - **Expected behavior**: What you expected to happen
      - **Actual behavior**: What actually happened
      - **Error messages**: Full error messages and stack traces if applicable
    </Accordion>
  </Step>
</Steps>

### Example Bug Report

<CodeGroup>
```typescript Minimal Reproduction
import { LinearRegression } from 'bun-scikit';

const X = [[1], [2], [3]];
const y = [2, 4, 6];

const model = new LinearRegression({ solver: 'normal' });
model.fit(X, y);

// Expected: predictions close to [8, 10]
// Actual: throws error
console.log(model.predict([[4], [5]]));
```

```bash Environment
Bun version: 1.3.9
bun-scikit version: 0.1.6
OS: Ubuntu 22.04
Node.js: 20.11.0 (for comparison)
```
</CodeGroup>

## Security Issues

<Warning>
  **Do not report security vulnerabilities through public GitHub issues.**
</Warning>

For security vulnerabilities:

<Steps>
  <Step title="Use Private Reporting">
    Follow the guidelines in `SECURITY.md` and use [GitHub's private vulnerability reporting](https://github.com/Seyamalam/bun-scikit/security/advisories/new).
  </Step>
  
  <Step title="Include Details">
    Provide:
    - Description of the vulnerability
    - Steps to reproduce
    - Potential impact
    - Suggested fix (if available)
  </Step>
  
  <Step title="Allow Response Time">
    We will respond to security reports within 48 hours and work with you to address the issue.
  </Step>
</Steps>
